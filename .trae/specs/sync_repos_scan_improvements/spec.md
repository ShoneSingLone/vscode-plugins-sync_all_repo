# Sync All Repos 插件 - 扫描功能改进

## Overview
- **Summary**: 改进 Sync All Repos 插件的扫描功能，实现默认不扫描、手动触发扫描、只扫描当前项目文件夹、并在单一表格中展示状态和结果
- **Purpose**: 优化用户体验，减少不必要的扫描操作，提高插件的针对性和效率
- **Target Users**: VS Code 用户，特别是需要管理多个 Git 仓库的开发者

## Goals
- 插件启动后默认不自动扫描仓库
- 用户打开界面后手动点击扫描按钮触发扫描
- 只扫描当前打开的项目文件夹，忽略其他项目
- 在单一表格中展示扫描结果和仓库状态
- 执行操作后实时更新状态和结果，不替代原有的扫描结果列表
- 确保执行操作不会对原有扫描结果列表产生任何干扰或替换
## Non-Goals (Out of Scope)
- 不修改现有的同步功能逻辑
- 不添加新的同步策略
- 不改变现有的配置存储方式
- 不支持多工作区扫描

## Background & Context
- 当前插件在启动时会自动扫描工作区中的所有仓库，可能导致不必要的性能消耗
- 用户希望更精确地控制扫描范围，只关注当前正在工作的项目
- 用户希望在同一界面完成所有操作，避免不必要的界面切换

## Functional Requirements
- **FR-1**: 插件启动时不自动扫描仓库
- **FR-2**: 提供手动扫描按钮，用户点击后开始扫描
- **FR-3**: 扫描范围限制为当前打开的项目文件夹
- **FR-4**: 在单一表格中展示扫描出的仓库列表和状态
- **FR-5**: 执行同步操作后自动更新状态和结果，不替代原有的扫描结果列表
- **FR-6**: 确保执行操作不会对原有扫描结果列表产生任何干扰或替换

## Non-Functional Requirements
- **NFR-1**: 扫描操作应在 3 秒内完成（对于标准大小的项目）
- **NFR-2**: 界面响应速度快，操作流畅
- **NFR-3**: 状态更新实时准确
- **NFR-4**: 代码修改最小化，保持现有功能完整性
- **NFR-5**: 详细的执行日志实时打印输出，便于测试反馈与问题排查
- **NFR-6**: 工作区文件夹显示逻辑不受配置中隐藏文件/文件夹设置的影响，严格根据物理文件夹是否为代码仓库来判断和显示

## Constraints
- **Technical**: 基于现有的 VS Code 扩展架构
- **Dependencies**: 依赖 Git 命令行工具
- **Platform**: 支持 Windows、macOS 和 Linux

## Assumptions
- 用户已打开至少一个包含 Git 仓库的项目文件夹
- Git 命令行工具已正确安装并配置
- 用户熟悉基本的 Git 操作

## Acceptance Criteria

### AC-1: 插件启动不自动扫描
- **Given**: 插件已安装并启用
- **When**: VS Code 启动或插件激活
- **Then**: 插件不会自动扫描仓库，界面显示空状态或提示用户手动扫描
- **Verification**: `human-judgment`
- **Notes**: 插件启动后应保持空闲状态，直到用户触发扫描

### AC-2: 手动扫描功能
- **Given**: 插件界面已打开
- **When**: 用户点击扫描按钮
- **Then**: 插件开始扫描当前项目文件夹
- **Verification**: `human-judgment`
- **Notes**: 扫描过程应有进度指示

### AC-3: 只扫描当前项目
- **Given**: 多个项目文件夹已打开
- **When**: 用户点击扫描按钮
- **Then**: 插件只扫描当前活动的项目文件夹，忽略其他项目
- **Verification**: `programmatic`
- **Notes**: 可通过检查扫描结果是否只包含当前项目文件夹来验证

### AC-4: 单一表格展示
- **Given**: 扫描完成
- **When**: 查看插件界面
- **Then**: 所有仓库信息和状态在单一表格中展示，无标签页切换
- **Verification**: `human-judgment`
- **Notes**: 表格应包含仓库名称、路径、分支、状态等信息

### AC-5: 状态实时更新
- **Given**: 仓库列表已展示
- **When**: 执行同步操作
- **Then**: 表格中的仓库状态和结果会实时更新
- **Verification**: `human-judgment`
- **Notes**: 操作过程中应有状态指示，完成后显示最终结果
